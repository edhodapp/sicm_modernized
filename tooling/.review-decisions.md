# SICM Modernized — Code-review decisions log

This file tracks every code-review finding raised against this
project, with explicit Ed-approved disposition (`**fix**`,
`**defer**`, or `**reject**`) and approval date. The
`tooling/scripts/check-review-decisions.sh` gate refuses commits
when any finding logged here lacks a disposition keyword and date.

The discipline (Ed's rule from 2026-04-26): triage decisions belong
to Ed. Claude surfaces findings with a recommendation but does not
unilaterally fix or defer them. See project CLAUDE.md
"Code-review discipline" for the full statement.

Append-only. Entries are never edited or removed; supersession
(e.g., a deferred item later fixed) is recorded as a new entry
under a later session block.

The format the gate parses: under any `## YYYY-MM-DD …` header,
every line beginning `- ` must contain one disposition keyword
(`**fix**`, `**defer**`, `**reject**`) followed by a `YYYY-MM-DD`
date. Free-form prose paragraphs (no leading `- `) are ignored.

Bypass: setting `SICM_BYPASS_REVIEW_DISPOSITIONS=1` in the
environment skips the gate for one invocation. Use only with
explicit conscious intent (file repair, log refactor) — the bypass
is logged on use.
